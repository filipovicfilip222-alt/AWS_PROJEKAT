1. Semantic search — kako radi
Šta je embedding (kratko)
Embedding je niz brojeva (vektor) koji predstavlja "značenje" teksta. Tekstovi sa sličnim značenjem imaju slične vektore.

Primer:

"Kako radi rekurzija?" → [0.12, -0.45, 0.78, ...] (1024 broja)
"Objasnite rekurzivni poziv" → [0.15, -0.42, 0.81, ...] (slične brojke)
"Šta je SQL?" → [-0.65, 0.23, -0.11, ...] (potpuno drugačije)
Cosine similarity meri "ugao" između vektora — vrednost između -1 i 1. Što bliže 1, to su tekstovi sličniji.

import boto3
import json

bedrock = boto3.client("bedrock-runtime", region_name="eu-central-1")
TITAN_MODEL_ID = "amazon.titan-embed-text-v2:0"

def generate_embedding(text: str) -> list[float]:
    """Generate 1024-dim embedding using Titan v2."""
    response = bedrock.invoke_model(
        modelId=TITAN_MODEL_ID,
        body=json.dumps({
            "inputText": text[:8000],  # max 8K karaktera
            "dimensions": 1024,
            "normalize": True  # već normalizovano za cosine
        })
    )
    body = json.loads(response["body"].read())
    return body["embedding"]

2.2 Nova Lambda: searchSemantic
backend/lambdas/search/semantic.py:

python
import numpy as np
from boto3.dynamodb.conditions import Key
from shared.response import api_handler, success_response
from shared.bedrock_client import generate_embedding
from shared.logger import logger
from shared.ddb_client import table

@api_handler
def handler(event, context):
    predmet = event["queryStringParameters"]["predmet"]
    query = event["queryStringParameters"]["q"]
    limit = int(event["queryStringParameters"].get("limit", 10))
    
    if len(query) < 3:
        return success_response(200, {"results": []})
    
    logger.info("Semantic search started", extra={"predmet": predmet, "queryLen": len(query)})
    
    # 1. Generiši embedding za query
    query_emb = np.array(generate_embedding(query))
    
    # 2. Pull sva pitanja za predmet (samo approved)
    response = table.query(
        IndexName="GSI5",  # nov GSI: pitanja po predmetu
        KeyConditionExpression=Key("GSI5PK").eq(f"PREDMET#{predmet}#APPROVED"),
        ProjectionExpression="questionId, pitanje, odgovor, tagovi, embedding, terminId, profesorIme"
    )
    questions = response.get("Items", [])
    
    if not questions:
        return success_response(200, {"results": []})
    
    # 3. Compute cosine similarity
    # (Titan je već normalizovan, pa je dot product = cosine similarity)
    embeddings = np.array([q["embedding"] for q in questions])
    scores = np.dot(embeddings, query_emb)
    
    # 4. Sortiraj i top-K
    top_indices = np.argsort(scores)[::-1][:limit]
    
    results = []
    for idx in top_indices:
        score = float(scores[idx])
        if score < 0.5:  # threshold za "relevantno"
            continue
        q = questions[idx]
        q.pop("embedding")  # ne treba u response-u
        q["score"] = round(score, 3)
        results.append(q)
    
    logger.info("Semantic search complete", extra={
        "totalScanned": len(questions),
        "returned": len(results)
    })
    
    return success_response(200, {"results": results})

5.1 Rate limiting po studentu
python
# Pri svakom AI chat pozivu
def check_rate_limit(student_id: str):
    today = datetime.utcnow().strftime("%Y-%m-%d")
    
    response = table.update_item(
        Key={"PK": f"RATELIMIT#{student_id}", "SK": f"AICHAT#{today}"},
        UpdateExpression="ADD #c :one SET #ttl = :ttl",
        ExpressionAttributeNames={"#c": "count", "#ttl": "ttl"},
        ExpressionAttributeValues={
            ":one": 1,
            ":ttl": int((datetime.utcnow() + timedelta(days=2)).timestamp())
        },
        ReturnValues="UPDATED_NEW"
    )
    count = response["Attributes"]["count"]
    
    if count > 20:  # max 20 AI pitanja po danu
        raise RateLimitError("Dnevni limit AI pitanja je 20. Pokušajte sutra ili zakažite konsultacije.")

7. ⚖️ Trade-offs — šta da znaš
Prednosti
✅ Velik UX skok. Studenti pitaju prirodnim jezikom. ✅ Jeftino za normalan volumen. Pod $5/mesec za fakultetski projekat. ✅ Reusable infrastructure. Embeddings se koriste i za "Slična pitanja" (#3), "Personalized path" (#9), itd. ✅ Demo-friendly. Pokazuje pravo razumevanje AI-a.

Mane
⚠️ Naivni semantic search radi do ~10K pitanja po predmetu. Posle, scan kroz DDB postaje spor (>1s). Tada:

Migracija na Aurora Serverless v2 + pgvector ($30+/mesec)
ili OpenSearch Serverless ($25/mesec base)
Ali za V1/V2 projekat nećeš ni blizu doći do 10K pitanja po predmetu.

⚠️ AI chat može halucinati. Mitigacija:

Eksplicitno tražiš confidence field
Material kao kontekst (RAG)
Discplay disclaimer u UI: "Odgovor generisan AI-em. Za precizne odgovore, zakaži konsultacije."
⚠️ Cold start. Prvi semantic search posle pause traje 2-3s. Posle je sub-second. Acceptable.

⚠️ Privacy. Studentska pitanja se logiraju (za rezime). Treba da bude jasno u UI da AI chat nije anoniman. Ili — anonimiziraj pri snimanju (samo question, ne studentId).

2. Design pravac — vizuelni identitet
